LORAVO
Privacy Policy
Last updated: 2026-02-14

Privacy, by design.

Loravo helps manage communications through connected services (Gmail, Outlook, Yahoo) and LXT AI features. Data remains user-controlled. Connections can be removed at any time.

Core promise: Personal information is not sold. Connected email data is not used to build advertising profiles. Provider data is accessed only when a connection is explicitly enabled and a requested feature requires it.

1) What Loravo is

Loravo is an application that can connect to third-party providers authorized by the user (such as Google/Gmail, Microsoft/Outlook, and Yahoo) to provide requested features — including inbox viewing, email tools, summaries, organization, and assistant-style workflows using LXT.

2) Information collected

Account & identifiers Required
  • Email address and/or phone number (depending on sign-in method).
  • Internal account ID (e.g., Firebase UID) to associate settings and connections.
Connected provider tokens OAuth
  • OAuth access/refresh tokens for Gmail, Outlook, or Yahoo only when a provider is connected.
  • Token expiry timestamps and connection status for reliability.
Content requested for features User-initiated
  • Email metadata/content only when loading inbox, opening a message, or generating a summary.
  • Messages typed into the app for chat or assistant workflows.
Optional context Optional
  • Location used only when enabled for local weather/news features.
  • Preferences (theme, settings) to personalize the experience.

3) How information is used

  • Provide features requested in-app (inbox, search, read, send, summaries, organization, alerts).
  • Secure access to connected providers and keep sessions reliable.
  • Maintain service performance, prevent abuse, and diagnose errors.

4) LXT AI and processing

LXT is the AI system used to generate summaries, suggested replies, and assistant-style workflows. AI processing runs only when a feature is explicitly triggered (for example: “summarize this email”).

  • Only the minimum necessary content is processed for the requested feature.
  • Provider tokens are stored securely and are not shared with unrelated parties.

5) Sharing

Personal information is not sold. Data is shared only in these situations:

  • With service infrastructure providers needed to operate the app (hosting, databases).
  • With third-party APIs that are explicitly connected (Gmail, Outlook, Yahoo) to perform requested actions.

6) Retention and control

  • Data is retained only as needed to provide the service.
  • A connected provider can be disconnected at any time, which revokes access for future requests.
  • Deletion requests can be submitted via the contact email below.
